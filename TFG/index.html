Documentación del TFG de Andrés Mena¶

En esta documentacion, se detallan los detalles de la implementación de la clase paciente, así como los módulos auxiliares utilizados para su implementación.

Modulo Paciente. Implementación¶

En este modulo se implementa la clase paciente, la cual se encargará de gestionar el procesado y cálculo de estadísticas de un paciente.

Se implementan distintas funciones para calcular los índices de apnea e hipopnea del paciente, así como para calcular estadísticas sobre el retraso que tienen las señales, y la correlación existente entre las mismas.

comprobar_validez_dataset(threshold=0.002) → bool¶

Comprueba que los datos del paciente sean válidado.

Se comprueba que todas las columnas indicadas como variables significativas al inicializar el Paciente contengan valores. Se tolera un threshold (porcentaje) de valores nan en la columna estudiada, y estos valores nan seran reemplazados por la última muestra válida de los datos.

Parámetros

threshold (float, optional) – Porcentaje de muestras que se tolera que esté con valor nan dentro de una columna. Defaults a 0.002.

Devuelve

True si el paciente se considera válido, al cumplir los requisitos del mínimo número de valores que no pueden ser nan. False en caso contrario.

Tipo del valor devuelto

bool

compute_sincronizacion_señales(seconds_prior=60, seconds_post=10, plot_results=False, signals=None, main_signal='spo2') → None¶

Calcula el retraso entre dos señales.

Para realizar el cálculo de la sincronización (el retraso) entre dos señales, se hace uso del módulo patient manager. Dicho retraso consiste en encontrar el desplazamiento necesario a aplicar a una señal frente a la otra para alcanzar el pico de correlación.

Parámetros

  • seconds_prior (float, optional) – Segundos previos sobre los que se va a desplazar la muestra para calcular la correlación. Defaults a 60.

  • seconds_post (float, optional) – Segundos posteriores sobre los que se va a desplazar la muestra para calcular la correlación. Defaults a 10.

  • plot_results (bool, optional) – Indica si se quiere mostrar el resultado gráficamente. Defaults a False.

  • signals (list, optional) – Lista de señales que se desea comparar contra la señal main_signal. Defaults a None, en cuyo caso se usan todas las señales significativas del paciente.

  • main_signal (str, optional) – Señal usada como referencia contra las que se comparan todas las señales del parámetro signals.

Ver también

patient_manager.plot_sincronizacion_temporal_simple().

Realiza el suavizado de la señal utilizando una ventana desalizante.

patient_manager.extract_stats_from_rs(rs, seconds_prior, seconds_post, time_step) → list¶

Extrae las estadísticas a partir de una lista con las correlaciones.

A partir de la lista de correlaciones, e indicando los segundos previos y posteriores, se calcula en qué posición se encuentra la máxima correlación, así como con qué retraso.

Parámetros

  • rs (list) – lista de valores de la correlación

  • time_step (float) – time_step con el espacio entre una muestra y la siguiente.

  • seconds_prior (float) – Segundos previos sobre los que se va a desplazar la muestra para calcular la correlación.

  • seconds_post (float) – Segundos posteriores sobre los que se va a desplazar la muestra para calcular la correlación.

Devuelve

Devuelve una lista con cinco elementos, donde cada uno indica: Posición del pico de correlación, pico de correlación, posición de la correlación sin desplazamiento, correlación sin desplazamiento, y el retraso de una señal respecto a la otra.

Tipo del valor devuelto

list

patient_manager.plot_sincronizacion_temporal_simple(rs, lag_list, column1, column2, time_step, seconds_prior, seconds_post, savefig=False) → None¶

Realiza la representación gráfica del retraso entre dos señales.

Se representa gráficamente el retraso. Utilizando los parámetros de la función, se especifican los resultados que se han obtenido al calcular la correlación conforme se desplaza la señal.

Parámetros

  • lag_list (list) – lista con los desplazamientos en los que se ha calculado la correlación.

  • column1 (str) – Primera columna de la que se quieren extraer los datos para calcular la sincronización.

  • column2 (str) – Segunda columna de la que se quieren extraer los datos para calcular la sincronización.

  • time_step (float) – time_step con el espacio entre una muestra y la siguiente.

  • seconds_prior (float) – Segundos previos sobre los que se va a desplazar la muestra para calcular la correlación.

  • seconds_post (float) – Segundos posteriores sobre los que se va a desplazar la muestra para calcular la correlación.

  • savefig (bool, optional) – Indica si se quiere almacenar el gráfico del resultado en un fichero pdf. Defaults a False.

Ver también

sincronizacion_temporal_simple(), extract_stats_from_rs().

patient_manager.sincronizacion_temporal_simple(df, time_step, column1, column2, seconds_prior=10, seconds_post=10, plot_results=True) → list¶

Calcula el retraso de una señal frente a otra.

Se calcula el retraso de una señal frente a otra. Utilizando los parámetros de la función, se puede especificar el rango de desplazamiento de las señales, así como cuánto se ha de desplazar cada una de las señales.

Parámetros

  • df (pd.DataFrame) – DataFrame que contiene la información de un paciente.

  • time_step (float) – time_step con el espacio entre una muestra y la siguiente.

  • column1 (str) – Primera columna de la que se quieren extraer los datos para calcular la sincronización.

  • column2 (str) – Segunda columna de la que se quieren extraer los datos para calcular la sincronización.

  • seconds_prior (float, optional) – Segundos previos sobre los que se va a desplazar la muestra para calcular la correlación. Defaults a 10.

  • seconds_post (float, optional) – Segundos posteriores sobre los que se va a desplazar la muestra para calcular la correlación. Defaults a 10.

  • plot_results (bool, optional) – Indica si se quiere mostrar el resultado gráficamente. Defaults a False.

Devuelve

Si se muestran los resultados gráficamente, no devuelve nada. En caso contrario, devuelve una lista con el primer elemento una lista de valores de la correlación, y el segundo elemento una lista con los desplazamientos en los que se ha calculado la correlación.

Tipo del valor devuelto

list

plot_sincronizacion_temporal_simple().
